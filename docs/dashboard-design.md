## Утверждённый порядок виджетов (сверху вниз)

### Блок 1: Карточки-счётчики (SummaryCards)
**Всегда показывать.** Первое, что видит пользователь — общая статистика системы.

Набор карточек в ряд (адаптивная сетка):
- Всего товаров → ссылка на `/product`
- Всего заказов → ссылка на `/orders`
- Всего пользователей → ссылка на `/users`
- Всего складов → ссылка на `/warehouses`
- Всего перемещений → ссылка на `/transfer`

Каждая карточка: иконка, число (крупно), подпись. Фон белый, border-radius 8px, тень.

**Источник:** `limit=1` к каждому списку, берём `totalCount`.

---

### Блок 2: Новые заказы
**Условный:** показывать только если есть заказы со статусом `new`.

| Поле | Описание |
|------|----------|
| Источник | `GET orders?limit=5` → фильтр `status === "new"` |
| Отобразить | Номер заказа, телефон, сумма, дата создания, метод оплаты |
| Действие | Ссылка на `/orders/edit/[id]` |
| Если пусто | Скрыть весь блок |

---

### Блок 3: Статистика продаж за месяц
**Всегда показывать.**

| Метрика | Расчет |
|---------|--------|
| Выручка (всего) | Сумма `total` по заказам со статусом `completed` за текущий месяц |
| Оплачено картой | Сумма `total` где `payment_method === "card"` |
| Оплачено наличными | Сумма `total` где `payment_method === "cash"` |
| Средний чек | `totalRevenue / completedOrdersCount` |
| Кол-во завершённых заказов | `completedOrdersCount` |
| Сумма скидок | Сумма `discount_total` |

**Источник:** `GET orders?limit=100` → фильтр `completed` и `created_at` в текущем месяце.

---

### Блок 4: Неотвеченные вопросы
**Условный:** показывать только если есть вопросы без ответа.

| Поле | Описание |
|------|----------|
| Источник | `GET product-question/all?limit=5` → фильтр `!is_answer` |
| Отобразить | Вопрос, имя товара (ссылка на товар), дата |
| Действие | Ссылка на `/product-questions` |
| Если пусто | Скрыть |

---

### Блок 5: Неотвеченные отзывы
**Условный:** показывать только если есть отзывы без ответа.

| Поле | Описание |
|------|----------|
| Источник | `GET product-review/all?limit=5` → фильтр `!is_answer` |
| Отобразить | Комментарий, рейтинг, имя товара, дата |
| Действие | Ссылка на `/product-reviews` |
| Если пусто | Скрыть |

---

### Блок 6: Активные перемещения (в пути)
**Условный:** показывать только если есть трансферы со статусом `processing`.

| Поле | Описание |
|------|----------|
| Источник | `GET transfers?limit=10` → фильтр `status === "processing"` |
| Отобразить | Тип (transfer/delivery), откуда→куда, дата создания |
| Действие | Ссылка на `/transfer/info/[id]` |
| Если пусто | Скрыть |

---

### Блок 7: Активные акции
**Условный:** показывать только если есть активные акции (`is_active === true` и `date_to` > сегодня).

| Поле | Описание |
|------|----------|
| Источник | `GET promotions?limit=10` → фильтр `is_active` и `date_to >= today` |
| Отобразить | Название, процент, дата начала, дата окончания |
| Действие | Ссылка на `/promotions` |
| Если пусто | Скрыть |

---

### Блок 8: Товары с малым остатком (≤ 5 шт)
**Условный:** показывать если есть товары с `available <= 5`.

| Поле | Описание |
|------|----------|
| Источник | `GET product/products?limit=100` → фильтр `available <= 5` |
| Отобразить | Название, код, остаток (`available`) |
| Действие | Ссылка на `/product/info/[id]` |
| Если пусто | Скрыть |

---

### Блок 9: SalesSchedule (задел)
**Место для будущего графика продаж.** Пока заглушка.

## Визуальный дизайн

**Стиль:** Сдержанный, соответствует текущей цветовой схеме проекта:
- Белый фон карточек, border-radius 8px
- Roboto, серые оттенки для текста
- Только семантические акценты: зелёный (success), красный (error), синий (#0075f2 для ссылок)
- Тени минимальные

### Адаптивность

| Breakpoint | Колонки | Поведение |
|-----------|---------|-----------|
| ≥ 1024px | 3 колонки для sales-stats | Карточки счётчиков в ряд |
| 768-1023px | 2 колонки | Карточки счётчиков 2+2+1 |
| < 768px | 1 колонка | Всё стекается вертикально |
