# Автозаполнение цен

## Описание

Админ настраивает правила автозаполнения цен на основе закупочной цены.

Для каждого **диапазона закупочной цены** задаётся **процент наценки** для каждого **типа цены**.

## Сущность "Диапазон автозаполнения"

| Поле       | Тип     | Описание                        |
| ---------- | ------- | ------------------------------- |
| id         | int     | ID диапазона                    |
| price_from | decimal | Начало диапазона (включительно) |
| price_to   | decimal | Конец диапазона                 |

### Правила

1. Диапазоны **не пересекаются** и **не имеют пробелов**
2. price_from и price_to — обязательные числовые поля
3. Сортировка: по возрастанию price_from на сервере

---

## Пример

### Типы цен (справочник)

| ID  | Название    |
| --- | ----------- |
| 1   | Розница     |
| 2   | Мелкий опт  |
| 3   | Крупный опт |

### Диапазоны автозаполнения

| ID  | От   | До   |
| --- | ---- | ---- |
| 1   | 0    | 99   |
| 2   | 100  | 499  |
| 3   | 500  | 999  |
| 4   | 1000 | 9999 |

### Матрица наценок

| Диапазон    | Розница | Мелкий опт | Крупный опт |
| ----------- | ------- | ---------- | ----------- |
| 0 - 99      | 100%    | 80%        | 70%         |
| 100 - 499   | 90%     | 70%        | 60%         |
| 500 - 999   | 80%     | 60%        | 50%         |
| 1000 - 9999 | 60%     | 45%        | 35%         |

---

## Расчёт цены

```
Закупочная: 50₽

Розница:     50 × (1 + 100%) = 50 × 2 = 100₽
Мелкий опт:  50 × (1 + 80%) = 50 × 1.8 = 90₽
Крупный опт: 50 × (1 + 70%) = 50 × 1.7 = 85₽ → округление вверх = 90₽
```

### Округление

- Всегда **вверх** до целого числа

## Страница автозаполнения цен

Отдельная страница `/price-auto-fill`.

### Элементы страницы

- **Поиск** — поиск по диапазону (например: "100-499")
- **Кнопка "Добавить диапазон"** — открывает модалку
- **Кнопка "Сохранить"** — зелёная кнопка для сохранения всех изменений процентов

### Структура таблицы

- **Вертикально (строки):** диапазоны автозаполнения
- **Горизонтально (колонки):** типы цен

|             | Розница | Мелкий опт | Крупный опт | Действия |
| ----------- | ------- | ---------- | ----------- | -------- |
| 0 - 99      | [100%]  | [80%]      | [70%]       | ✏️ 🗑️    |
| 100 - 499   | [90%]   | [70%]      | [60%]       | ✏️ 🗑️    |
| 500 - 999   | [80%]   | [60%]      | [50%]       | ✏️ 🗑️    |
| 1000 - 9999 | [60%]   | [45%]      | [35%]       | ✏️ 🗑️    |

**[Сохранить]** — зелёная кнопка рядом с поиском. Сохраняет все изменённые проценты.

### Операции

| Операция         | Описание                                     |
| ---------------- | -------------------------------------------- |
| Поиск            | Поиск по диапазону                           |
| Создать диапазон | Кнопка "+ Добавить диапазон" → модалка       |
| Редактировать    | Клик на ✏️ → модалка с полями "от", "до"     |
| Удалить          | Клик на 🗑️ → подтверждение                   |
| Изменить %       | Изменение значения в инпуте                  |
| Сохранить        | Кнопка "Сохранить" — сохраняет все изменения |

**Пагинации нет** — все диапазоны на одной странице, чтобы случайно не сбросить изменённые поля.

### Модалка "Добавить/Редактировать диапазон"

| Поле | Тип    | Описание         |
| ---- | ------ | ---------------- |
| От   | number | Начало диапазона |
| До   | number | Конец диапазона  |

## API эндпоинты

### Диапазоны

| Метод  | Путь                       | Описание          |
| ------ | -------------------------- | ----------------- |
| GET    | /api/price-fill/ranges     | Список диапазонов |
| POST   | /api/price-fill/ranges     | Создать диапазон  |
| PATCH  | /api/price-fill/ranges/:id | Обновить диапазон |
| DELETE | /api/price-fill/ranges/:id | Удалить диапазон  |

### Правила (проценты)

| Метод  | Путь                      | Описание         |
| ------ | ------------------------- | ---------------- |
| GET    | /api/price-fill/rules     | Список правил    |
| POST   | /api/price-fill/rules     | Создать правило  |
| PATCH  | /api/price-fill/rules/:id | Обновить процент |
| DELETE | /api/price-fill/rules/:id | Удалить правило  |

---

## Валидация

1. **price_from >= 0**
2. **price_to > price_from**
3. При создании: проверить, что новый диапазон не пересекается с существующими
