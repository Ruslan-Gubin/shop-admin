# Модуль: Характеристики товаров

## Контекст

**Цель:** Добавить к товарам текстовые характеристики (label → value).
**Бизнес-модель:** E-commerce с доставкой, розница/маркетплейс.
**Подход:** Справочник имён характеристик — позволяет фильтровать товары по характеристикам.

---

## Структура данных

### 1. Справочник имён характеристик (FeatureName)

Хранится отдельно, чтобы можно было фильтровать товары по одним и тем же характеристикам.

```typescript
interface FeatureName {
  id: number;
  name: string;           // "Материал", "Срок годности", "Количество страниц"
  slug: string;           // "material", "shelf-life", "page-count" (для URL/фильтра)
  unit?: string;          // Опционально: "дней", "листов", "мм" (пока не используется)
  category_id?: number;   // FK → Category (опционально, можно ограничить категорией)
  is_active: boolean;
  sort_order: number;     // Порядок сортировки в карточке товара
  created_at: Date;
  updated_at: Date;
}
```

**Примеры:**
```
id: 1, name: "Материал", slug: "material"
id: 2, name: "Срок годности", slug: "shelf-life"
id: 3, name: "Количество страниц", slug: "page-count"
id: 4, name: "Запах", slug: "scent"
id: 5, name: "Тип лампы", slug: "lamp-type"
```

### 2. Значения характеристик товара (ProductFeature)

Привязка к конкретному товару.

```typescript
interface ProductFeature {
  id: number;
  product_id: number;      // FK → Product
  feature_name_id: number; // FK → FeatureName
  value: string;           // "Пластик", "3 дня", "18 листов"
  sort_order: number;      // Порядок отображения в карточке товара
  created_at: Date;
  updated_at: Date;
}
```

**Примеры:**
```
product_id: 5, feature_name_id: 1, value: "Пластик"
product_id: 5, feature_name_id: 2, value: "3 дня"
product_id: 12, feature_name_id: 5, value: "Светодиод"
product_id: 12, feature_name_id: 4, value: "Морская свежесть"
```

## API эндпоинты

### FeatureName (справочник)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/feature-name` | Список всех имён характеристик |
| GET | `/feature-name/{id}` | Получить одно имя характеристики |
| POST | `/feature-name` | Создать новое имя характеристики |
| PATCH | `/feature-name/{id}` | Обновить имя характеристики |
| DELETE | `/feature-name/{id}` | Удалить имя характеристики |

### ProductFeature (значения)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/product-feature?product_id={id}` | Получить все характеристики товара |
| POST | `/product-feature` | Добавить характеристику товару |
| PATCH | `/product-feature/{id}` | Обновить значение характеристики |
| DELETE | `/product-feature/{id}` | Удалить характеристику товара |
| POST | `/product-feature/bulk` | Массовое создание/обновление характеристик |

## Страницы админки

### 1. `/feature-names` — Справочник имён характеристик

**Назначение:** CRUD для имён характеристик (название + slug).

**UI-элементы:**
- Таблица имён характеристик (desktop + mobile)
- Фильтр: категория (опционально), статус (активные/архивные)
- Модалка создания/редактирования
- Кнопка удаления

**Форма создания/редактирования:**
```
Название: [Материал________]
Slug: [material_________] (автогенерация из названия)
Категория: [Выберите категорию___] (опционально)
Активен: [✓]
```

---

### 2. `/products/[id]/features` — Характеристики товара (опциональная страница)

**Назначение:** Просмотр/редактирование всех характеристик конкретного товара.

**UI-элементы:**
- Список характеристик товара
- Добавление новой характеристики (select + input)
- Редактирование значения
- Удаление характеристики

---

### 3. Блок характеристик на странице редактирования товара

**Назначение:** Добавление/редактирование характеристик直接在 странице товара.

**UI-элементы:**
- Список характеристик (label → value)
- Кнопка "Добавить характеристику"
- Форма добавления: select (имя хар-ки) + input (значение)
- Inline-редактирование значения
- Кнопка удаления

## Взаимосвязь с товаром

### Вариант A: Блок на странице редактирования товара (IMPLEMENTED NOW)

Характеристики добавляются直接在 форме редактирования товара.

**User flow:**
1. Админ открывает `/product/edit/5`
2. Видит блок "Характеристики" под ценами
3. Нажимает "Добавить характеристику"
4. Выбирает из справочника → вводит значение
5. Сохраняет товар

**Плюсы:** Всё в одном месте.
**Минусы:** Неудобно редактировать несколько товаров с одинаковой характеристикой.

### Вариант B: Отдельная страница `/products/[id]/features`

Характеристики редактируются на отдельной странице.

**Плюсы:** Чистая страница только для хар-к.
**Минусы:** Лишний клик для редактирования.

## Детали реализации

### 1. При создании товара

На странице `/product/create` добавить блок "Характеристики":
- Возможность добавить несколько характеристик
- Select (имя хар-ки из справочника) + Input (значение)
- Кнопка "Добавить ещё"
- Кнопка "Удалить" для каждой строки

### 2. При редактировании товара

На странице `/product/edit/[id]`:
- Загружать текущие характеристики товара
- Позволять добавлять/редактировать/удалять
- Сохранять изменения вместе с товаром

### 3. При удалении имени характеристики

- Проверить, есть ли связанные ProductFeature
- Если есть — предупредить админа
- При подтверждении — удалить всё связанное (CASCADE)

### 4. Фильтрация по характеристикам (future)

После реализации справочника можно добавить:
- Фильтр товаров по характеристикам (e.g., "Материал = Пластик")
- В справочнике: `category_id` для ограничения хар-к по категории
