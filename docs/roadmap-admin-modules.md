# Roadmap: Модули админки для интернет-магазина

## Контекст проекта

**Бизнес-модель:** E-commerce с доставкой (физические товары)
**Тип магазина:** Розница / Маркетплейс
**Склады:** Мультисклад (несколько локаций)

**Уже реализовано:**
- Аутентификация (sign-in, sign-up)
- Каталог товаров (список, создание, редактирование)
- Категории товаров
- Типы цен
- Автозаполнение цен
- Промо-акции
- Скидки на корзину
- Пользователи (CRUD)

## Детальное описание модулей

### 1. Склады (Warehouses)

**Назначение:** Справочник складов для мультискладского учёта.

**Сущность:**
```
Warehouse {
  id: number
  name: string           // "Склад Москва"
  address: string         // "г. Москва, ул. Примерная, 1"
  is_active: boolean      // Активен / В архиве
  created_at: Date
  updated_at: Date
}
```

**CRUD операции:** Создание, редактирование, удаление, список.

**UI-требования:**
- Таблица складов (desktop + mobile)
- Модальное окно создания/редактирования
- Фильтр по статусу (активные / архивные)

---

### 2. Остатки на складах (Inventory / Stock)

**Назначение:** Связь товар ↔ склад ↔ количество.

**Сущность:**
```
InventoryStock {
  id: number
  warehouse_id: number   // FK → Warehouse
  product_id: number      // FK → Product
  quantity: number        // Количество на складе
  reserved: number        // Зарезервировано (в корзинах/заказах)
  available: number       // quantity - reserved (вычисляемое)
  updated_at: Date
}
```

**CRUD операции:**
- Просмотр остатков товара по всем складам
- Редактирование количества (ручное)
- Списание / Приёмка
- История движений (in/out)

**UI-требования:**
- Таблица "Товар → Склад → Остаток"
- Фильтр по складу
- Поиск по товару
- История изменений остатков

---

### 3. Заказы (Orders)

**Назначение:** Приём и управление заказами.

**Сущность:**
```
Order {
  id: number
  order_number: string    // "ORD-2024-000001"
  customer_id: number     // FK → Customer
  status: OrderStatus     // Enum: new, processing, shipped, delivered, cancelled
  items: OrderItem[]      // Товары в заказе
  
  subtotal: number        // Сумма товаров
  discount: number       // Скидка на корзину
  shipping_cost: number   // Стоимость доставки
  total: number           // Итого
  
  shipping_address: Address
  notes: string           // Заметки к заказу
  
  created_at: Date
  updated_at: Date
}

OrderItem {
  id: number
  product_id: number
  product_name: string    // Snapshot названия на момент заказа
  price_type: PriceType
  price: number           // Snapshot цены
  quantity: number
  total: number
}
```

**CRUD операции:**
- Создание заказа (админ может создать вручную)
- Просмотр списка заказов
- Детали заказа
- Изменение статуса
- Добавление товаров в заказ
- Отмена заказа

**UI-требования:**
- Список заказов с фильтрами (статус, дата, клиент)
- Детальная страница заказа
- Добавление товаров в заказ
- Смена статуса (с комментарием)

---

### 4. Статусы заказов (Order Statuses)

**Назначение:** Управление жизненным циклом заказа.

**Flow:**
```
new → processing → shipped → delivered
                ↘ cancelled
```

**UI-требования:**
- Блок смены статуса на странице заказа
- История смены статусов (кто, когда, комментарий)
- Уведомление клиенту при смене статуса (опционально)

---

### 5. Клиенты (Customers)

**Назначение:** Профили покупателей.

**Сущность:**
```
Customer {
  id: number
  user_id?: number        // FK → User (если зарегистрирован)
  
  name: string
  email: string
  phone: string
  
  addresses: Address[]
  
  orders_count: number   // Количество заказов
  total_spent: number    // Общая сумма покупок
  
  created_at: Date
  updated_at: Date
}
```

**CRUD операции:**
- Список клиентов (гости + зарегистрированные)
- Профиль клиента
- История заказов клиента
- Редактирование данных

**UI-требования:**
- Таблица клиентов
- Карточка клиента с историей заказов
- Фильтр по активности (новые, постоянные)

---

### 6. Адреса доставки (Addresses)

**Назначение:** Сохранение адресов клиентов.

**Сущность:**
```
Address {
  id: number
  customer_id: number
  
  name: string           // "Дом", "Работа"
  city: string
  street: string
  house: string
  apartment?: string
  postal_code?: string
  comment?: string
  
  is_default: boolean
}
```

**UI-требования:**
- Блок "Адреса доставки" в карточке клиента
- CRUD для адресов
- Выбор адреса по умолчанию

---

### 7. Корзины (Carts)

**Назначение:** Управление корзинами клиентов.

**Сущность:**
```
Cart {
  id: number
  customer_id?: number   // null для гостей
  session_id: string     // Для гостей
  
  items: CartItem[]
  
  applied_discount?: CartDiscount
  
  expires_at: Date       // Срок жизни корзины
  created_at: Date
  updated_at: Date
}

CartItem {
  id: number
  product_id: number
  price_type: PriceType
  quantity: number
  price_snapshot: number
}
```

**UI-требования:**
- Просмотр abandoned carts (брошенные корзины)
- Фильтр по дате, стоимости
- Восстановление корзины клиента

---

### 8. Доставки (Shipping)

**Назначение:** Управление способами доставки.

**Сущность:**
```
ShippingMethod {
  id: number
  name: string           // "Курьер", "СДЭК", "Самовывоз"
  description: string
  price: number          // Фиксированная цена или 0
  is_active: boolean
  
  // Интеграция (опционально)
  api_key?: string
  tracking_url_template?: string
}

ShippingRate {
  id: number
  shipping_method_id: number
  warehouse_id?: number  // Если мульти-точка отправки
  zone?: string          // Зона доставки
  min_weight?: number
  max_weight?: number
  price: number
}
```

**UI-требования:**
- Справочник способов доставки
- Настройка тарифов
- Интеграция с трекингом (опционально)

## Архитектурные заметки

### Интеграция с существующим кодом

Все новые модули должны следовать архитектурному паттерну текущего проекта:

```
src/app/{module}/
├── page.tsx              # Server Component (получение данных)
├── action.ts             # Server Actions (мутации)
├── schema.ts            # Zod схемы валидации
├── components/
│   ├── {Module}TableWrapper/   # Client Component (обёртка)
│   └── modal-{module}-form/    # Модалки форм
└── {Module}.module.css
```

### Переиспользуемые виджеты

Использовать существующие виджеты:
- `MainTable` — desktop таблица
- `MainMobileTable` — mobile таблица (с cookie-sync паттерном)
- `Modal` — модальные окна
- `InputMain`, `Select`, `ButtonMain` — UI примитивы

### State management

- Zustand для глобального состояния (меню, уведомления)
- React state (`useState`) для локального состояния форм
- Server Components для получения данных (async/await)

### Стилизация

- CSS Modules (`.module.css`)
- Глобальные стили в `src/app/styles/`
